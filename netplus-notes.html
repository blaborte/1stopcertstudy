1.0 Networking Concepts 23%
🔗 The OSI Reference Model ▼
The OSI model is the backbone of CompTIA networking questions. Every protocol, device, and troubleshooting step maps to one of its seven layers. Memorize layer names, PDU names, and what operates at each layer.
The 7 Layers
  • Layer 7 – Application: User-facing protocols. HTTP, HTTPS, FTP, SMTP, DNS, SNMP, LDAP, RDP. PDU = Data.
  • Layer 6 – Presentation: Data formatting, encryption/decryption, compression. SSL/TLS negotiation starts here. JPEG, MPEG, ASCII. PDU = Data.
  • Layer 5 – Session: Establishes, manages, terminates sessions. NetBIOS, RPC, PPTP, L2TP. Controls half/full duplex. PDU = Data.
  • Layer 4 – Transport: End-to-end communication, segmentation, port numbers. TCP (reliable) and UDP (fast). PDU = Segment (TCP) / Datagram (UDP).
  • Layer 3 – Network: Logical addressing (IP) and routing between networks. Routers live here. IP, ICMP, OSPF, BGP, RIP. PDU = Packet.
  • Layer 2 – Data Link: MAC addressing, framing, error detection (CRC). Switches, bridges, NICs. 802.3, 802.11, STP, ARP. PDU = Frame.
  • Layer 1 – Physical: Raw bit transmission. Cables, hubs, repeaters, signals, connectors. PDU = Bit.
💡
Mnemonic top-down (7→1): All People Seem To Need Data Processing
Mnemonic bottom-up (1→7): Please Do Not Throw Sausage Pizza Away

📡 TCP vs UDP — Transport Layer Protocols ▼
TCP and UDP are the two Transport layer workhorses. Every exam scenario involving reliability vs. speed maps back to this comparison.
TCP — Transmission Control Protocol
  • Connection-oriented: establishes a session before data transfer
  • Three-way handshake: SYN → SYN-ACK → ACK
  • Reliable delivery: acknowledgments, retransmission of lost packets
  • Ordered delivery: sequence numbers ensure correct reassembly
  • Flow control: windowing prevents sender from overwhelming receiver
  • Use cases: HTTP/S, FTP, SSH, Telnet, SMTP, RDP — anything where accuracy matters
UDP — User Datagram Protocol
  • Connectionless: no handshake, fire and forget
  • No reliability guarantees: no ACKs, no retransmission
  • Low overhead: smaller header (8 bytes vs TCP's 20+ bytes)
  • Use cases: DNS, DHCP, SNMP, VoIP, video streaming, TFTP, NTP — speed over reliability
🔑
Key distinction: TCP adds reliability at the cost of overhead. UDP trades reliability for speed and efficiency. VoIP uses UDP because a dropped packet is better than delayed audio.

🌐 IPv4 Addressing & Subnetting ▼
Subnetting appears on nearly every network exam. Master the CIDR table and the math behind hosts per subnet, and you will never miss these questions.
Address Classes
  • Class A: 1–126.x.x.x — /8 default — 16 million hosts — large enterprises
  • Class B: 128–191.x.x.x — /16 default — 65,534 hosts — medium networks
  • Class C: 192–223.x.x.x — /24 default — 254 hosts — small networks
  • Class D: 224–239.x.x.x — Multicast (no subnet)
  • Class E: 240–255.x.x.x — Reserved / Research
  • Loopback: 127.0.0.1 — tests local TCP/IP stack
  • APIPA: 169.254.x.x — self-assigned when DHCP fails
RFC 1918 Private Ranges
  • 10.0.0.0/8 — Class A private (10.0.0.0 – 10.255.255.255)
  • 172.16.0.0/12 — Class B private (172.16.0.0 – 172.31.255.255)
  • 192.168.0.0/16 — Class C private (192.168.0.0 – 192.168.255.255)

☁️ Cloud Concepts & Connectivity ▼
Cloud is a growing portion of the N10-009 exam. Know the service models, deployment models, and cloud-specific networking features.
Service Models
  • SaaS (Software as a Service): vendor manages everything — infrastructure, OS, runtime, app. Customer just uses software. Examples: Gmail, Office 365, Salesforce.
  • PaaS (Platform as a Service): vendor manages OS and infrastructure. Customer manages applications. Examples: Heroku, Google App Engine.
  • IaaS (Infrastructure as a Service): vendor manages hardware. Customer manages OS, apps, data. Examples: AWS EC2, Azure VMs.
Deployment Models
  • Public: shared infrastructure owned by cloud provider. Multi-tenant. Cost-efficient.
  • Private: dedicated infrastructure owned/operated by single org. More control, more cost.
  • Hybrid: combines on-premises and public cloud. Sensitive data on-prem, burst to cloud.
  • Community: shared infrastructure among orgs with common requirements (same industry, compliance).
Key Cloud Networking Concepts
  • VPC (Virtual Private Cloud): isolated network segment within public cloud
  • NFV (Network Functions Virtualization): replacing physical network devices with software
  • SD-WAN: software-controlled WAN with zero-touch provisioning and central policy management
  • SASE: Secure Access Service Edge — converges networking and security into cloud service
  • ZTA (Zero Trust Architecture): never trust, always verify — least privilege access everywhere
🔑
Shared Responsibility: In SaaS the provider is responsible for nearly everything. In IaaS the customer has the most responsibility. The line shifts depending on the service model.
🌍 IPv6 & Modern Network Environments ▼
IPv6 is tested on the N10-009 with increasing frequency. Know address types, shortening rules, and transition mechanisms.
IPv6 Basics
  • 128-bit addresses written as 8 groups of 4 hex digits separated by colons
  • Shortening rules: omit leading zeros in a group; use :: once to replace consecutive all-zero groups
  • Global unicast: 2000::/3 — equivalent to public IPv4, routable on internet
  • Link-local: FE80::/10 — auto-configured, not routable, only valid on local segment
  • Loopback: ::1 — equivalent to IPv4 127.0.0.1
  • Multicast: FF00::/8 — replaces IPv4 broadcast
Transition Mechanisms
  • Dual Stack: device runs both IPv4 and IPv6 simultaneously — most common approach
  • Tunneling (6to4): encapsulates IPv6 packets inside IPv4 for transport over IPv4 infrastructure
  • NAT64: translates between IPv6 and IPv4 networks
🎯
Exam Focus: IPv6 does NOT natively use NAT — it has enough address space. IPv6 uses SLAAC for automatic configuration (Stateless Address Autoconfiguration).
2.0 Network Implementation 20%
🔄 VLANs & Switching ▼
VLANs are logical network segments within a physical switch. They are critical for segmentation, security, and reducing broadcast domains. Trunking allows multiple VLANs to travel between switches on a single link.
VLAN Fundamentals
  • VLAN: logical broadcast domain created within a switch — devices on different VLANs cannot communicate without a router (Layer 3)
  • Access port: carries traffic for only ONE VLAN — connects to end devices (PCs, phones)
  • Trunk port: carries traffic for MULTIPLE VLANs — connects switches to switches or switch to router
  • 802.1Q tagging: IEEE standard adding 4-byte VLAN tag to Ethernet frames on trunk ports
  • Native VLAN: the one VLAN whose traffic is sent UNTAGGED on a trunk port — default is VLAN 1, change it for security
  • Voice VLAN: separate VLAN for VoIP traffic to ensure QoS priority
  • SVI (Switch Virtual Interface): Layer 3 interface on a VLAN used for inter-VLAN routing on multilayer switches
Spanning Tree Protocol (STP)
  • Purpose: prevents Layer 2 loops by blocking redundant paths until needed
  • Root bridge: switch with lowest Bridge ID (priority + MAC) — all traffic flows through it
  • Port states: Blocking → Listening → Learning → Forwarding (802.1D) — RSTP converges much faster
  • Port roles: Root port (best path to root), Designated port (forwarding on segment), Alternate/Blocked port
  • RSTP (802.1w): Rapid Spanning Tree — much faster convergence than classic STP
  • PortFast: skips STP states for access ports connecting to end devices
  • BPDU Guard: disables a PortFast port if it receives a BPDU — prevents rogue switches
Link Aggregation
  • LACP (802.3ad): IEEE open standard — combines multiple physical links into one logical link
  • Benefits: increased bandwidth + redundancy — if one link fails, others carry traffic

Frequency & Channel Planning
  • 2.4GHz: 3 non-overlapping channels (1, 6, 11) — more range, more interference, slower
  • 5GHz: many non-overlapping channels — less range, less interference, faster
  • 6GHz (Wi-Fi 6E): newest band — cleanest spectrum, shortest range
  • Channel overlap: adjacent channels interfere — always use non-overlapping channels on nearby APs
  • Band steering: automatically moves capable clients to 5GHz to reduce 2.4GHz congestion
Wireless Security
  • WPA2-Personal: AES/CCMP encryption, single pre-shared key — home and small business
  • WPA2-Enterprise: 802.1X authentication via RADIUS — unique per-user credentials, enterprise
  • WPA3: SAE (Simultaneous Authentication of Equals) handshake — resistant to dictionary attacks, forward secrecy
  • WEP: deprecated and broken — never use
  • SSID suppression: hides network from casual scanning — weak security, clients still reveal SSID in probe requests
AP Architecture
  • Autonomous AP: standalone, self-configured — simple deployments
  • Lightweight AP: managed by a WLC (Wireless LAN Controller) via CAPWAP tunnels — enterprise deployments
  • Cloud-managed AP: managed via cloud dashboard (e.g., Cisco Meraki) — no on-prem controller needed
🛣️ Routing Technologies ▼
Routing is how packets find their way across networks. Know static vs dynamic routing, the major protocols, and how routers make forwarding decisions.
Routing Basics
  • Static routing: manually configured routes — no overhead, best for small/simple networks
  • Default route: 0.0.0.0/0 — the gateway of last resort for all unmatched traffic
  • Longest prefix match: router chooses the most specific matching route — /28 beats /24 beats /0
  • Administrative distance: trustworthiness of a route source — lower = more trusted

Dynamic Routing Protocols
  • RIP (v1/v2): Distance vector, hop count metric, max 15 hops — simple but limited. UDP port 520.
  • OSPF: Link-state, Dijkstra algorithm, no hop count limit — scales well, fast convergence. Areas reduce complexity.
  • EIGRP: Cisco's hybrid protocol — composite metric (bandwidth + delay). Fast, efficient, complex.
  • BGP: Path vector, the internet's routing protocol — connects autonomous systems. Policy-based routing.
NAT & PAT
  • NAT: Network Address Translation — maps private IPs to public IPs for internet access
  • PAT (NAT Overload): many private IPs share ONE public IP using different port numbers — most common form
  • FHRP: First Hop Redundancy Protocol — HSRP (Cisco), VRRP (open), GLBP (load balancing) — provides default gateway redundancy
🏗️ Physical Installation & Infrastructure ▼
Physical layer knowledge is directly tested. Know the rack terminology, cabling standards, connector types, and environmental requirements for data centers.
Distribution Frames
  • MDF (Main Distribution Frame): primary wiring nexus — backbone switches, demarcation points, ISP connections
  • IDF (Intermediate Distribution Frame): floor/area-level distribution — connects horizontal cabling to MDF backbone
Cable Types & Standards
  • Cat 5e: 1 Gbps up to 100m — most common legacy standard
  • Cat 6: 1 Gbps (100m) or 10 Gbps (55m) — improved crosstalk resistance
  • Cat 6A: 10 Gbps up to 100m — augmented, for 10GBaseT full distance
  • Cat 8: 25/40 Gbps up to 30m — data center use, shielded required
  • Single-mode fiber (SMF): laser light source, tiny core (~9μm), spans kilometers — WAN/campus backbone
  • Multimode fiber (MMF): LED light source, larger core (50/62.5μm), up to ~550m — building/campus
  • Plenum cable: fire-retardant jacket for use in air-handling spaces (drop ceilings, raised floors)
  • DAC (Direct Attach Copper): twinaxial cable with SFP connectors — short-range high-speed connections in racks
Fiber Connectors
  • LC: Local Connector — small form factor, push-pull, most common in modern deployments
  • SC: Subscriber Connector — square push-pull, older but still common
  • ST: Straight Tip — bayonet twist-lock, older legacy deployments
  • MPO: Multi-fiber Push On — 12 or 24 fiber ribbon cable, data center high-density
PoE Standards
  • 802.3af (PoE): up to 15.4W per port — IP phones, basic APs, cameras
  • 802.3at (PoE+): up to 30W per port — more powerful APs, PTZ cameras
  • 802.3bt (PoE++): up to 60–100W per port — laptops, video conferencing systems
3.0 Network Operations 19%
📋 Documentation & Change Management ▼
Network documentation is the foundation of operational excellence. The N10-009 tests your ability to understand what each document type contains and when to use each.
Documentation Types
  • Physical diagram: actual physical connections — where cables run, rack positions, patch panel ports
  • Logical diagram: IP addresses, VLANs, subnets, routing paths — how traffic flows logically
  • Rack diagram: equipment in racks — unit positions, power, cable management
  • Cable map/diagram: documents every cable — from, to, type, length, label
  • Wireless heat map: RF coverage visualization — shows signal strength across floor plan
  • IPAM (IP Address Management): tracks IP allocation — prevents conflicts, enables auditing
  • Asset inventory: hardware, software, licensing, warranty data — compliance and planning
  • SLA (Service Level Agreement): defines guaranteed service levels between provider and customer
Change Management Process
  • RFC (Request for Change): formal documentation of a proposed change — what, why, when, how
  • Impact analysis: what could break? who is affected? what is the rollback plan?
  • CAB (Change Advisory Board): reviews and approves changes — authorizes downtime and maintenance windows
  • Maintenance window: approved time period for making changes — minimizes business impact
  • Rollback plan: how to undo the change if it fails — must be documented BEFORE implementing
  • Configuration baseline: known-good configuration snapshot — the golden config to restore to
🎯
Exam Focus: Change management is about control and documentation — not speed. The CAB approves changes, not implements them. Always have a rollback plan before touching production.
📊 Network Monitoring & SNMP ▼
Monitoring is how you know your network is healthy before users call to complain. SNMP, syslog, and flow data are the three pillars tested on the exam.
SNMP Versions
  • SNMPv1: original — community strings (plaintext), basic functionality
  • SNMPv2c: community strings (still plaintext) but adds bulk operations and improved error handling
  • SNMPv3: adds authentication (MD5/SHA) and encryption (DES/AES) — the only secure version
  • Community strings: passwords for v1/v2c — read-only vs read-write — never use defaults
  • MIB (Management Information Base): database of manageable device objects, each identified by OID
  • Traps: unsolicited alerts from device to manager — device reports events proactively (port 162)
Syslog
  • Protocol: UDP port 514 — sends log messages from devices to a central collector
  • Severity levels 0–7: Emergency(0), Alert(1), Critical(2), Error(3), Warning(4), Notice(5), Info(6), Debug(7)
  • SIEM: Security Information and Event Management — collects, correlates, and analyzes syslog and event data
Network Flow & Packet Capture
  • NetFlow / sFlow / IPFIX: metadata about flows (who talked to whom, how much, when) — not full packet capture
  • Packet capture (PCAP): full packet contents — Wireshark, tcpdump — used for deep analysis
  • Port mirroring (SPAN): copies switch port traffic to a monitoring port — passive, no inline impact
  • Baseline metrics: normal behavior measurements — deviations trigger anomaly alerts
🔧 DNS, DHCP & Time Services ▼
DNS and DHCP are the invisible plumbing of every network. If either fails, users cannot work. Know how both work end-to-end and how to troubleshoot them.
DHCP Process — DORA
  • Discover: client broadcasts on UDP 68→67 — "Is there a DHCP server?"
  • Offer: server responds with an available IP address
  • Request: client formally requests the offered IP
  • Acknowledge: server confirms and delivers IP, mask, gateway, DNS, lease time
  • IP helper / DHCP relay: router forwards DHCP broadcasts between subnets (ip helper-address command)
  • Scope: range of IPs available for assignment
  • Reservations: permanent IP-to-MAC binding — printers, servers
  • Exclusions: IPs in the scope range not to be assigned
  • SLAAC: IPv6 Stateless Address Autoconfiguration — devices build their own addresses without DHCP

🆘 Disaster Recovery Concepts ▼
DR planning is about keeping promises to your organization about how fast you can recover from failure. Every DR concept maps to a specific measurement or site type.
Key DR Metrics
RPO
Recovery Point Objective — max acceptable data LOSS measured in time. Determines backup frequency. "We can afford to lose 4 hours of data."
RTO
Recovery Time Objective — max acceptable DOWNTIME. Determines how fast systems must be restored. "We must be back online in 2 hours."
MTTR
Mean Time To Repair — average time to fix a failed component after it breaks. Lower = better reliability.
MTBF
Mean Time Between Failures — average operational time between failures. Higher = more reliable hardware.
DR Site Types
  • Hot site: fully operational mirror of production — real-time data replication, can fail over in minutes. Most expensive.
  • Warm site: hardware present, partially configured, NO current data — can be operational in hours. Middle cost.
  • Cold site: facility only, no equipment — cheapest, days to weeks for recovery.
High Availability
  • Active-Active: all nodes process traffic simultaneously — provides load balancing AND fault tolerance
  • Active-Passive: standby node takes over only when primary fails — no load balancing benefit
  • FHRP: First Hop Redundancy Protocol — HSRP/VRRP/GLBP provide default gateway redundancy
💡
Remember: RPO = how much DATA you can lose. RTO = how much TIME you can be down. In a BCP conversation: short RPO needs frequent backups; short RTO needs a hot site.
4.0 Network Security 14%
🛡️ Security Fundamentals & CIA Triad ▼
All of network security is built on the CIA triad. Every attack, control, and policy maps back to one or more of these three properties. Understand them deeply.
The CIA Triad
  • Confidentiality: data is accessible ONLY to authorized parties. Controls: encryption, access control lists, classification. Threats: eavesdropping, data theft, social engineering.
  • Integrity: data has NOT been altered without authorization. Controls: hashing (MD5/SHA), digital signatures, checksums. Threats: man-in-the-middle, data corruption.
  • Availability: authorized users CAN access systems when needed. Controls: redundancy, backups, DDoS mitigation, HA. Threats: DoS/DDoS, ransomware, hardware failure.
Common Security Terminology
Vulnerability
A weakness in a system that can be exploited — unpatched software, misconfiguration, weak passwords
Threat
A potential event or actor that could exploit a vulnerability — malware, attacker, natural disaster
Exploit
Actual code or technique that takes advantage of a vulnerability
Risk
The combination of likelihood × impact — what you manage through security controls
Honeypot
Decoy system designed to attract and trap attackers — provides threat intelligence
Honeynet
Network of honeypots — broader deception environment for studying attacker behavior
Defense in Depth
  • Layered security: multiple overlapping controls — if one fails, others compensate
  • Network segmentation: VLANs, DMZ, firewalls — contain breaches, limit lateral movement
  • Zero Trust Architecture (ZTA): never trust, always verify — every access request authenticated regardless of location
⚔️ Common Network Attacks ▼
Knowing attack types is mandatory — you need to recognize them in scenario questions and know the appropriate countermeasures.
Denial of Service Attacks
  • DoS: single attacker floods target to exhaust resources (CPU, memory, bandwidth, connections)
  • DDoS: coordinated attack from thousands of compromised hosts (botnet) — much harder to block
  • Mitigations: rate limiting, IP blackholing, traffic scrubbing services, overprovisioning bandwidth
Layer 2 & Network Attacks
  • ARP poisoning/spoofing: attacker sends fake ARP replies — poisons ARP cache to intercept traffic (Man-in-the-Middle)
  • MAC flooding: floods switch CAM table with fake MACs — forces switch to broadcast all traffic (like a hub)
  • VLAN hopping: double-tagging or switch spoofing to access VLANs — prevented by not using VLAN 1 and disabling DTP
  • DNS poisoning: corrupts DNS cache to redirect users to malicious sites
  • Rogue DHCP server: attacker runs unauthorized DHCP — assigns wrong gateway/DNS for MitM attacks
  • Evil twin / Rogue AP: fake wireless AP impersonating a legitimate one — captures credentials
Social Engineering Attacks
  • Phishing: fraudulent emails trick users into revealing credentials or installing malware
  • Spear phishing: targeted phishing at specific individuals with personalized content
  • Whaling: spear phishing targeting executives (C-suite)
  • Vishing: voice/phone-based phishing
  • Smishing: SMS-based phishing
  • Tailgating/Piggybacking: following authorized person through secure entrance
  • Shoulder surfing: watching someone type credentials
  • Dumpster diving: recovering sensitive information from discarded materials
🎯
Exam scenario tip: If a question mentions intercepting traffic between two hosts on the same network — think ARP poisoning. If it mentions flooding switch ports — MAC flooding.
🔐 Authentication & Access Control ▼
Who can access what, and how do we verify their identity? These are the core questions access control answers. Know the protocols, models, and MFA factors.
Identity & Access Management (IAM)
  • Authentication: proving you are who you claim to be (verifying identity)
  • Authorization: what you are permitted to do after being authenticated
  • Accounting: tracking what you did (audit trails, logs)
  • MFA factors: Something you know (password/PIN) + Something you have (token/smartcard) + Something you are (biometrics)
  • SSO (Single Sign-On): authenticate once, access multiple resources — SAML, OAuth, OIDC
AAA Protocols
  • RADIUS: UDP-based, combines authentication and authorization — used for network access (Wi-Fi, VPN, dial-up)
  • TACACS+: TCP-based, separates authentication/authorization/accounting — used for device administration (Cisco routers/switches)
  • 802.1X: port-based NAC — device must authenticate via RADIUS before gaining network access. Supplicant → Authenticator (switch/AP) → RADIUS server
  • LDAP: Lightweight Directory Access Protocol (port 389/636) — queries Active Directory / directory services
  • SAML: Security Assertion Markup Language — XML-based federation for SSO between organizations
Encryption & PKI
  • Data in transit: TLS 1.2/1.3 (HTTPS, SMTPS), IPSec (VPN)
  • Data at rest: AES-256 full disk encryption, BitLocker, FileVault
  • Symmetric encryption: same key for encrypt and decrypt — fast (AES) — key distribution problem
  • Asymmetric encryption: public key encrypts, private key decrypts — secure key exchange (RSA, ECC)
  • PKI: Public Key Infrastructure — CA issues certificates binding public keys to identities
  • Digital signatures: private key signs, public key verifies — provides authenticity and non-repudiation
🔒 Network Security Controls ▼
Controls are the mechanisms we implement to protect networks. Know the difference between each type and when to apply them.
Firewall Types
  • Packet filtering: inspects IP headers (src/dst IP, port, protocol) — stateless, fast, limited
  • Stateful inspection: tracks connection state — knows if a packet is part of an established session
  • NGFW (Next-Generation Firewall): deep packet inspection, application awareness, IPS, SSL inspection
  • WAF (Web Application Firewall): specifically protects web apps — filters HTTP/S for XSS, SQLi, CSRF
IDS vs IPS
  • IDS (Intrusion Detection System): passive — monitors and ALERTS. Does NOT block. Can be placed out-of-band via port mirror.
  • IPS (Intrusion Prevention System): inline — monitors AND BLOCKS malicious traffic automatically. Must be inline.
  • HIDS: Host-based IDS — monitors a single host (logs, file changes, processes)
  • NIDS: Network-based IDS — monitors network traffic via port mirror or TAP
  • Signature-based: matches known attack patterns — fast, misses zero-days
  • Anomaly-based: detects deviations from baseline — catches unknowns, higher false positives
Network Access Control & Hardening
  • NAC (Network Access Control): enforces security policy before granting network access — checks patch level, AV status
  • Port security: limits MAC addresses per switch port — shutdown/restrict/protect violation modes
  • DHCP snooping: switch feature that drops DHCP offers from untrusted ports — blocks rogue DHCP servers
  • Dynamic ARP Inspection (DAI): validates ARP packets against DHCP snooping table — prevents ARP poisoning
  • ACL (Access Control List): rule-based packet filtering on routers and firewalls — permit/deny by IP and port
  • Device hardening: change defaults, disable unused ports/services, apply patches, enable logging
  • Screened subnet (DMZ): network segment for public-facing servers — separated from internal network by firewalls
5.0 Network Troubleshooting 24%
🔍 Troubleshooting Methodology ▼
CompTIA's seven-step troubleshooting methodology is not just theoretical — it is directly tested with scenario questions. Know the exact order and what each step involves.
The 7 Steps
  • 1. Identify the problem: gather information, question users, identify symptoms, check what changed, duplicate if possible
  • 2. Establish a theory of probable cause: question the obvious, consider OSI top-down or bottom-up, divide and conquer
  • 3. Test the theory to determine the cause: if confirmed → plan resolution; if not → new theory or escalate
  • 4. Establish a plan of action: document steps, identify potential side effects, prepare rollback
  • 5. Implement the solution or escalate: make the change, follow the plan
  • 6. Verify full system functionality: confirm fix works, implement preventive measures
  • 7. Document findings, actions, outcomes: update runbooks, knowledge base, change records
OSI Troubleshooting Approaches
  • Bottom-up (L1→L7): start at Physical layer — check cables, then switch, then router, then application. Best for unknown problems.
  • Top-down (L7→L1): start at Application — check the app/service, then configuration, then network. Best when user reports app issue.
  • Divide and conquer: start at Layer 3 (ping) — if that works, go up; if not, go down. Efficient for experienced techs.
🎯
Exam strategy: Scenario says "user cannot connect to website but can ping the IP" → DNS issue (L7). "Cannot ping anything" → check physical/link first. "One user affected, not others" → isolate to that user's device/port.

Hardware Tools
  • Cable tester: verifies cable continuity, identifies opens, shorts, split pairs, miswires
  • Toner & probe: toner generates tone on one end, probe detects it at other — traces cable runs through walls/ceilings
  • Butt set: clips to punchdown block pins — tests telephone circuits, places calls
  • OTDR (Optical Time Domain Reflectometer): tests fiber — locates breaks, splices, bends by measuring light reflection
  • Wi-Fi analyzer: scans wireless spectrum — shows channels, signal strength, interference sources
  • Visual fault locator: injects red laser into fiber — visible light shows breaks or sharp bends
  • Multimeter: measures voltage, current, resistance in copper cables
💡
PBQ tip: Cable fault questions: short = signal on 2 pins from 1 source. Open = no continuity. Split pair = wires from different pairs twisted together — passes continuity but causes crosstalk.
📡 Wireless Troubleshooting ▼
Wireless problems are some of the most common real-world issues and heavily tested on the exam. Know the causes, symptoms, and solutions for each wireless issue type.
Common Wireless Issues
  • Channel overlap / co-channel interference: adjacent or same-channel APs interfere — redesign channel plan, use non-overlapping channels (1,6,11)
  • Signal degradation: physical obstructions (walls, metal, elevators), distance — reposition AP or add more APs
  • Insufficient coverage: dead zones — add APs or use mesh networking
  • 2.4GHz interference: microwave ovens, Bluetooth, baby monitors, cordless phones — move to 5GHz band
  • Client disassociation: sudden drops — check for RF interference, deauth attacks (if systematic)
  • Roaming misconfiguration: client sticks to far AP instead of closer one — configure roaming thresholds, use band steering
  • Hidden SSID: SSID not broadcasting — must manually type SSID to connect; still discoverable via packet capture
  • Wrong security protocol: cannot connect — ensure client supports same WPA version as AP
🎯
Exam scenario: "Some laptops cannot connect after switching to 5GHz" — older 802.11g/b adapters only support 2.4GHz. Fix: configure AP for dual-band, not 5GHz-only.
⚡ Performance Issues & Cable Faults ▼
Performance problems and cable issues are the "plumber" side of networking — less glamorous but heavily tested and critical in real deployments.
Performance Issues
  • Latency: delay in packet delivery — check for congested links, long routing paths, overloaded devices
  • Jitter: variable latency — especially harmful to VoIP/video. Fix: QoS prioritization
  • Packet loss: packets not arriving — check for CRC errors, congestion, duplex mismatch
  • Throughput vs bandwidth: bandwidth is max capacity; throughput is actual achieved rate — always less due to overhead and congestion
  • Bottleneck: single point limiting overall performance — often a slow WAN link or oversubscribed switch uplink
  • Duplex mismatch: one side full-duplex, other half-duplex — causes intermittent connectivity, CRC errors, and slowness
Cable Faults
  • Attenuation: signal weakens over distance — exceeding cable length limits (100m for UTP)
  • Crosstalk: signal bleed from adjacent wire pairs — NEXT (Near-End) is most common, caused by poor termination
  • EMI/RFI: electromagnetic/radio frequency interference from motors, fluorescent lights — use STP cable or reroute
  • Open: break in cable — no continuity — cable tester shows no signal
  • Short: two conductors touching — signal appears on both wires
  • Split pair: wires from different pairs swapped — passes continuity test but causes high crosstalk
  • TX/RX reversed: transmit and receive wires swapped — link fails or uses MDIX to auto-correct
Interface Issues
  • CRC errors: corrupted frames — physical problem (bad cable, EMI, duplex mismatch)
  • Runts: frames smaller than 64 bytes — collision or duplex mismatch
  • Giants: frames larger than 1518 bytes — mismatched MTU, jumbo frame misconfiguration
  • Error disabled: switch disabled port due to security violation (port security) or STP BPDU Guard trigger
